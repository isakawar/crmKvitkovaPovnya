# План фіксу багів MVP проекту

## 🐛 Відомі баги та побажання

### 1. ✅ Валідація custom_amount для власного розміру
- **Проблема**: При створенні замовлення з розміром "Власний" поле `custom_amount` не валідується правильно
- **Статус**: Виправлено
- **Зміни**: 
  - Додано валідацію в `app/blueprints/orders/routes.py`
  - Покращено обробку в `app/services/order_service.py`
  - Додано логування в JavaScript

### 2. 🔴 Критичні баги (Те що точно треба)

#### 2.1 Не виводиться сума власного букета
- **Проблема**: Сума власного букета не відображається в списку замовлень
- **Статус**: Потребує виправлення
- **Потрібно**: Додати відображення `custom_amount` в таблиці замовлень

#### 2.2 Поле "Власна сума" в модалці редагування
- **Проблема**: В модалці редагування замовлення немає поля "Власна сума"
- **Статус**: Потребує виправлення
- **Потрібно**: Додати поле `custom_amount` в форму редагування

#### 2.3 Фільтрація по даті - тільки конкретна дата
- **Проблема**: Не можна обрати період, тільки конкретну дату
- **Статус**: Потребує виправлення
- **Потрібно**: Додати можливість вибору діапазону дат

#### 2.4 Можливість додавати та працювати зі статусами
- **Проблема**: Немає можливості керувати статусами доставок
- **Статус**: Потребує реалізації
- **Потрібно**: Створити систему управління статусами

#### 2.5 Можливість видалення доставок
- **Проблема**: Немає можливості видаляти доставки
- **Статус**: Потребує реалізації
- **Потрібно**: Додати функціонал видалення доставок

#### 2.6 ✅ Продовження підписки - поле "дата першої доставки"
- **Проблема**: При продовженні підписки з'являється поле "дата першої доставки" яке потрібно прибрати
- **Статус**: Виправлено
- **Зміни**: 
  - Прибрано поле "дата першої доставки" з `extend_order_modal.html`
  - Виправлено JavaScript для використання правильного API `/deliveries/{id}/extend-subscription-with-form`
  - Створено новий маршрут який приймає дані з форми і правильно обробляє продовження
  - Виправлено конфлікт у `deliveries_list.html` де старий код перезаписував обробник форми

#### 2.7 ✅ Тести для критичної функціональності
- **Проблема**: Відсутні тести для основної функціональності, особливо продовження підписки та custom_amount
- **Статус**: Реалізовано
- **Зміни**:
  - Створено `tests/test_delivery_extension_fixed.py` - тести продовження підписки
  - Створено `tests/test_orders.py` - тести створення замовлень
  - Оновлено `tests/test_services/test_order_service.py` - тести сервісів
  - Виправлено `tests/conftest.py` для роботи з поточним кодом
  - **Результати**: 10 пройдених тестів, 4 з редиректами (нормальна поведінка)

### 3. 🟢 Побажання (хотєлка)

#### 3.1 Клік по імені клієнта
- **Побажання**: При натисканні на ім'я клієнта відкривати список доставок з датами
- **Статус**: Побажання
- **Потрібно**: Додати інтерактивність до імені клієнта

#### 3.2 Номер доставки клієнта
- **Побажання**: Виводити номер доставки саме цього клієнта для визначення листівки з рекомендаціями
- **Статус**: Побажання
- **Потрібно**: Додати відображення порядкового номера доставки

### 3. 📱 Проблеми з мобільною версією
- **Проблема**: Інтерфейс не адаптивний для мобільних пристроїв
- **Статус**: Потребує аналізу
- **Потрібно перевірити**:
  - Таблиці на мобільних пристроях
  - Форми створення/редагування
  - Навігація

### 4. 🔐 Проблеми з безпекою
- **Проблема**: Відсутній CSRF захист
- **Статус**: Критично
- **Потрібно**:
  - Додати CSRF токени до форм
  - Налаштувати Flask-WTF

### 5. 📊 Проблеми з даними
- **Проблема**: Можливі проблеми з цілісністю даних
- **Статус**: Потребує перевірки
- **Потрібно перевірити**:
  - Зв'язки між таблицями
  - Каскадне видалення
  - Унікальність даних

### 6. ⚡ Проблеми з продуктивністю
- **Проблема**: Повільні запити при великій кількості даних
- **Статус**: Потребує аналізу
- **Потрібно перевірити**:
  - Індекси в базі даних
  - Оптимізація запитів
  - Пагінація

### 7. 🎨 Проблеми з UI/UX
- **Проблема**: Непослідовний дизайн
- **Статус**: Потребує аналізу
- **Потрібно перевірити**:
  - Кольорова схема
  - Розміри шрифтів
  - Відступи та відстані
