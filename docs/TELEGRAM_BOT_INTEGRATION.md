# 🤖 ІНТЕГРАЦІЯ З TELEGRAM БОТОМ

## 🎯 ГОЛОВНА МЕТА
Створити Telegram бота для автоматизації роботи кур'єрів з можливістю:
- Отримання списку доставок
- Відмітки про виконання доставок
- Автоматичного оновлення статусів в CRM
- Списання коштів з клієнтів
- Оновлення статистики кур'єрів

---

## 📋 ДЕКОМПОЗИЦІЯ ЗАДАЧ

### 🔧 ЕТАП 1: ПІДГОТОВКА ІНФРАСТРУКТУРИ

#### 1.1 Налаштування Telegram Bot API
- [ ] Створити бота через @BotFather
- [ ] Отримати Bot Token
- [ ] Додати токен в конфігурацію додатку
- [ ] Встановити необхідні бібліотеки (python-telegram-bot)

#### 1.2 Структура проекту
- [ ] Створити папку `app/telegram_bot/`
- [ ] Створити файли:
  - [ ] `__init__.py`
  - [ ] `bot.py` - основний клас бота
  - [ ] `handlers.py` - обробники команд
  - [ ] `keyboards.py` - клавіатури для інтерфейсу
  - [ ] `utils.py` - допоміжні функції
  - [ ] `notifications.py` - відправка повідомлень

#### 1.3 Конфігурація
- [ ] Додати Telegram settings в `config.py`
- [ ] Створити middleware для бота
- [ ] Налаштувати логування для бота

---

### 🗃️ ЕТАП 2: РОЗШИРЕННЯ МОДЕЛЕЙ ДАНИХ

#### 2.1 Модель Courier
- [ ] Додати поле `telegram_chat_id` (зв'язок з Telegram користувачем)
- [ ] Додати поле `telegram_username` (опціонально)
- [ ] Додати поле `bot_notifications_enabled` (увімкнені сповіщення)

#### 2.2 Модель Delivery
- [ ] Додати поле `telegram_message_id` (ID повідомлення в Telegram)
- [ ] Додати поле `delivered_at` (час доставки)
- [ ] Додати поле `delivered_by_telegram` (чи була доставка відмічена через бота)

#### 2.3 Модель Client (для списання коштів)
- [ ] Переглянути логіку credits - як вони списуються
- [ ] Додати поле `credits_history` або окрему таблицю для історії транзакцій

#### 2.4 Міграції
- [ ] Створити міграцію для `telegram_chat_id` в `courier`
- [ ] Створити міграцію для `telegram_message_id` та `delivered_at` в `delivery`
- [ ] Створити міграцію для історії кредитів (якщо потрібно)

## 🛠️ ТЕХНОЛОГІЇ

### Backend
- **Python Telegram Bot API** - для роботи з Telegram
- **APScheduler** - для cron завдань  
- **SQLAlchemy** - розширення моделей
- **Flask** - інтеграція з існуючим додатком

### Telegram
- **Inline Keyboards** - для інтерактивних кнопок
- **Callback Queries** - для обробки натискань
- **Message Formatting** - для красивого відображення

### Безпека
- **Webhook Secret** - для захисту webhook
- **Rate Limiting** - захист від спаму
- **Authorization** - перевірка доступу кур'єрів
