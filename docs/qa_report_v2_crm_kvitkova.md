  2. Critical Flows — Повні user flows (E2E)                  
                                                                                                                                                                                                                                                    
  Flow 1: Нова підписка (найважливіший flow)                  
                                                                                                                                                                                                                                                    
  1. Менеджер відкриває /subscriptions → [Нова підписка]
  2. Заповнює форму:                                                                                                                                                                                                                                
     - client_instagram (пошук існуючого або новий)           
     - recipient_name, recipient_phone                                                                                                                                                                                                              
     - city, street, building_number, floor, entrance                                                                                                                                                                                               
     - delivery_method: courier / nova_poshta / pickup                                                                                                                                                                                              
     - size: S / M / L                                                                                                                                                                                                                              
     - delivery_day: Пн/Вт/.../Нд                                                                                                                                                                                                                   
     - first_delivery_date (перша доставка)                   
     - for_whom, bouquet_type, preferences                                                                                                                                                                                                          
     - discount (%)                                                                                                                                                                                                                                 
  3. POST /subscriptions/draft → створює Subscription{status=draft}                                                                                                                                                                                 
  4. Менеджер підтверджує → POST /subscriptions/<id>/extend або /orders/new                                                                                                                                                                         
     → subscription_service.create_subscription()                                                                                                                                                                                                   
     → генерує 4 Order + 4 Delivery                                                                                                                                                                                                                 
     → Delivery.status = 'Очікує'                                                                                                                                                                                                                   
  5. У день доставки менеджер відкриває /route-generator                                                                                                                                                                                            
     → вибирає дату → бачить Очікує доставки                                                                                                                                                                                                        
     → route_optimizer_service.optimize_json() → розраховує маршрут                                                                                                                                                                                 
     → зберігає DeliveryRoute{status=draft}                                                                                                                                                                                                         
  6. Менеджер → /routes → призначає кур'єра                                                                                                                                                                                                         
     → POST /routes/<id>/assign-and-send                                                                                                                                                                                                            
     → Telegram bot надсилає маршрут кур'єру                                                                                                                                                                                                        
     → Delivery.status = 'Розподілено'                                                                                                                                                                                                              
  7. Кур'єр у Telegram: приймає маршрут                                                                                                                                                                                                             
     → DeliveryRoute.status = 'accepted', accepted_at = now()                                                                                                                                                                                       
  8. Кур'єр виконує доставку → натискає "✅ Доставлено"                                                                                                                                                                                             
     → Delivery.status = 'Доставлено', delivered_at = now()                                                                                                                                                                                         
     → courier.deliveries_count += 1                                                                                                                                                                                                                
     → (❌ BUG: client.credits НЕ списуються)                                                                                                                                                                                                       
  9. Після 4 доставок менеджер продовжує підписку                                                                                                                                                                                                   
     → POST /orders/<id>/extend-subscription                                                                                                                                                                                                        
     → ще 4 Delivery                                                                                                                                                                                                                                
                                                                                                                                                                                                                                                    
  Непокриті перевірки в цьому flow:                           
  - Що якщо delivery_day не збігається з first_delivery_date? (наприклад, day=Пн, date=субота)                                                                                                                                                      
  - Що відбувається при extend якщо старі доставки ще не завершені?                                                                                                                                                                                 
  - Чи рахуються знижки при розрахунку ціни?                       
  - Чи блокується зміна підписки після призначення маршруту?                                                                                                                                                                                        
                                                                                                                                                                                                                                                    
  ---                                                                                                                                                                                                                                               
  Flow 2: Одноразове замовлення з сертифікатом                                                                                                                                                                                                      
                                                                                                                                                                                                                                                    
  1. POST /certificates/create → Certificate{status=active, code=unique}
  2. Менеджер приймає замовлення клієнта                                                                                                                                                                                                            
  3. POST /orders/new → Order + Delivery                      
  4. POST /certificates/validate {code} → перевіряє активність                                                                                                                                                                                      
  5. Certificate прив'язується до Order → Certificate.status = 'used', used_at = now()                                                                                                                                                              
  6. Знижка/безкоштовний розмір застосовується до замовлення                                                                                                                                                                                        
  7. Доставка проходить той самий маршрут що й Flow 1 (кроки 5-8)                                                                                                                                                                                   
                                                                                                                                                                                                                                                    
  Непокриті:                                                                                                                                                                                                                                        
  - Чи можна застосувати прострочений сертифікат? (expires_at <= today — баг #7 з коду)                                                                                                                                                             
  - Чи можна використати сертифікат двічі? (атомарність status='used')                                                                                                                                                                              
  - Сертифікат типу subscription — як він реалізований?               
                                                                                                                                                                                                                                                    
  ---                                                                                                                                                                                                                                               
  Flow 3: CSV Import → автоматичне створення підписок                                                                                                                                                                                               
                                                                                                                                                                                                                                                    
  1. Менеджер готує CSV з полями (Ім'я, Телефон, Місто, День, Розмір, ...)
  2. GET /import/kvitkovapovnya → форма                                                                                                                                                                                                             
  3. POST /import/kvitkovapovnya/preview → preview з помилками/попередженнями                                                                                                                                                                       
  4. Менеджер переглядає → виправляє CSV якщо треба                                                                                                                                                                                                 
  5. POST /import/kvitkovapovnya/execute                                                                                                                                                                                                            
     - якщо delivery_count == 0 → одноразове замовлення                                                                                                                                                                                             
     - якщо delivery_count is null → тільки клієнт                                                                                                                                                                                                  
     - якщо delivery_count >= 5 → extend існуючої підписки                                                                                                                                                                                          
     - else → нова підписка з N доставками                                                                                                                                                                                                          
  6. Результат: N клієнтів, M підписок, K замовлень                                                                                                                                                                                                 
                                                                                                                                                                                                                                                    
  Непокриті:                                                  
  - Що якщо той самий Instagram зустрічається у CSV двічі?                                                                                                                                                                                          
  - Що якщо телефон є в базі, але Instagram новий (інший клієнт)?                                                                                                                                                                                   
  - Rollback при частковій помилці: чи створені до помилки записи відкочуються?
  - Максимальний розмір CSV (тест DoS/memory)?

Менеджер отримує підтвердження

  - Що якщо action_id підробити (чужого менеджера)?                                                                                                                                                                                                 
  - Чи перевіряє агент дозволи при виконанні (re-auth)?
  - 50k символів у запиті → агент відповів 200 (DoS неможливий rate-limit)

  3. Data Risks — Ризики цілісності даних                                                                                                                                                                                                           
                                                                                                                                                                                                                                                    
  DR-1. 🔴 ПІДТВЕРДЖЕНО: Race condition при створенні клієнтів
                                                                                                                                                                                                                                                    
  Тест-результат:                                                                                                                                                                                                                                   
  5 concurrent POST /clients/new (same instagram) → [200, 200, 200, 400, 200]                                                                                                                                                                       
  3 з 5 запитів повернули 200 → в базі може бути 3 дублікати клієнта з однаковим instagram.                                                                                                                                                         
                                                                                           
  Механізм: _validate_contact_fields() робить SELECT → перевіряє → INSERT. Між SELECT і INSERT інший запит може пройти те саме.                                                                                                                     
                                                                                                                                                                                                                                                    
  Виправлення:                                                                                                                                                                                                                                      
  # Модель:                                                                                                                                                                                                                                         
  instagram = db.Column(db.String(128), unique=True, nullable=True, index=True)
  # + на рівні БД: UNIQUE constraint                                                                                                                                                                                                                
  # + в коді: try/except IntegrityError                                                                                                                                                                                                             
                                                                                                                                                                                                                                                    
  ---                                                                                                                                                                                                                                               
  DR-2. 🔴 ПІДТВЕРДЖЕНО: Клієнти доступні без авторизації                                                                                                                                                                                           
                                                                                                                                                                                                                                                    
  Тест-результат: GET /clients/json → 200 без cookies. Повертає 992 клієнти з полями id, instagram, name, phone, telegram.
                                                                                                                                                                                                                                                    
  Наслідок: Повна база клієнтів (992 записи) доступна будь-кому хто знає URL.

  DR-3. 🔴 Orphan Deliveries при видаленні замовлення         
                                                                                                                                                                                                                                                    
  POST /orders/<id>/delete каскадно видаляє deliveries, але:
  - Якщо доставка вже в DeliveryRoute → RouteDelivery запис може залишитись                                                                                                                                                                         
  - Маршрут кур'єра міститиме "фантомну" доставку                                                                                                                                                                                                   
                                                                                                                                                                                                                                                    
  Перевірка: order_service.delete_order() — чи є видалення RouteDelivery записів?

  DR-4. 🔴 Підписка без delivery_count логіки в CSV                                                                                                                                                                                                 
                                                                                                                                                                                                                                                    
  if delivery_count >= 5:                
      → extend_subscription()                                                                                                                                                                                                                       
  Але якщо в базі немає активної підписки для цього клієнта — extend_subscription() впаде або створить некоректний стан.                                                                                                                            
                                                                                                                                                                                                                                                    
  ---                                                                                                                                                                                                                                               
  DR-5. 🟡 Delivery без Route = невидима для кур'єра                                                                                                                                                                                                
                                                                                                                                                                                                                                                    
  Delivery зі статусом 'Очікує' що не потрапила в жоден DeliveryRoute — не буде надіслана кур'єру. Немає UI-попередження "ця доставка не в жодному маршруті".                                                                                       
                                                                                                                                                                                                                                                    
  ---                                                         
  DR-6. 🟡 Subscription reschedule — часткове застосування                                                                                                                                                                                          
                                                                                                                                                                                                                                                    
  apply_reschedule_plan() переносить pending deliveries. Якщо після часткового застосування відбудеться exception:
  - Частина доставок перенесена, частина — ні                                                                                                                                                                                                       
  - Немає транзакційного rollback на рівні всього плану                                                                                                                                                                                             
                                                                                                                                                                                                                                                    
  ---                                                                                                                                                                                                                                               
  DR-7. 🟡 Courier.deliveries_count десинхронізується         
                                                                                                                                                                                                                                                    
  deliveries_count інкрементується при set_delivery_status('Доставлено'). Але:
  - Якщо статус виставляється через /florist/deliveries/status — чи також інкрементується?                                                                                                                                                          
  - Якщо статус змінюється назад з Доставлено на Очікує — deliveries_count не декрементується                                                                                                                                                       
                                                                                                                                                                                                                                                    
  ---                                                                                                                                                                                                                                               
  DR-8. 🟡 Transaction без прив'язки до клієнта = "в нікуди"  
                                                                                                                                                                                                                                                    
  Transaction.client_id — nullable. Можна створити транзакцію без клієнта. Фінансова аналітика буде неповною.
                                                                                                                                                                                                                                                    
  ---                                                         
  DR-9. 🟡 Certificate використано, але замовлення видалено                                                                                                                                                                                         
                                                                                                                                                                                                                                                    
  Certificate.order_id nullable. Якщо замовлення видалене → certificate.order_id = NULL, але certificate.status = 'used'. Неможливо дізнатись до якого замовлення відносився сертифікат.
                                                                                                                                                                                                                                                    
  ---                                                         
  DR-10. 🟡 cached_result_json в DeliveryRoute — stale cache                                                                                                                                                                                        
                                                                                                                                                                                                                                                    
  route_optimizer_service кешує результат маршруту в cached_result_json. Якщо після кешування додати/видалити доставку — маршрут залишиться старим. Немає інвалідації кешу.
