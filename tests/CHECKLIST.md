Що перевіряють юніт тести                                                                                                                                                   
                                                  
  ClientService                                                                                                                                                               
   
  - Створення клієнта — позитивний флоу (нормалізація @ в instagram)                                                                                                          
  - Створення клієнта без instagram → помилка                        
  - Невалідний телефон (без +380, занадто короткий/довгий, літери) → помилка                                                                                                  
  - Валідний телефон +380XXXXXXXXX → успіх                                                                                                                                    
  - Дублікат instagram → структурована помилка з client_id                                                                                                                    
  - Дублікат instagram з @ префіксом (@handle = handle) → помилка                                                                                                             
  - Дублікат telegram → структурована помилка                                                                                                                                 
  - Дублікат телефону → структурована помилка                                                                                                                                 
  - Пошук клієнта по instagram                                                                                                                                                
  - Пошук клієнта по телефону                                                                                                                                                 
                                                                     
  SubscriptionService — розрахунок дат                                                                                                                                        
                                                                     
  - Weekly: наступна дата рівно +7 днів, той самий день тижня                                                                                                                 
  - Weekly: послідовність 4 дат — кожен інтервал рівно 7 днів        
  - Bi-weekly: базова дата +14 днів, прив'язка до дня тижня                                                                                                                   
  - Bi-weekly: дрейф snap не перевищує 20 днів (будь-який день тижня)                                                                                                         
  - Bi-weekly: послідовність — всі 4 дати однаковий день тижня                                                                                                                
  - Monthly: наступна дата поблизу +28 днів (відхилення ≤ 3 дні)                                                                                                              
  - Monthly: snap до найближчого потрібного дня тижня                                                                                                                         
  - build_delivery_dates завжди повертає 4 дати                                                                                                                               
  - build_delivery_dates — перша дата = вхідна дата                                                                                                                           
  - build_delivery_dates_n — повертає довільну кількість дат                                                                                                                  
  - build_delivery_dates_n(0) → порожній список                                                                                                                               
  - Reschedule Weekly: якщо gap ≤ 4 дні — зсув на повний тиждень вперед                                                                                                       
  - Reschedule Bi-weekly: якщо gap ≤ 9 днів — зсув на 14 днів вперед                                                                                                          
                                                                                                                                                                              
  DeliveryService                                                                                                                                                             
                                                                                                                                                                              
  - Фінансовий тиждень починається в суботу, закінчується в п'ятницю                                                                                                          
  - Фінансовий тиждень містить сьогоднішню дату
  - offset +1 → наступний фінансовий тиждень (+7 днів)                                                                                                                        
  - Групування доставок по даті — правильна кількість у кожній групі                                                                                                          
  - Розподілено → Доставлено — інкрементує courier.deliveries_count                                                                                                           
  - Очікує → Доставлено — не інкрементує лічильник (документує поточну поведінку)                                                                                             
  - delivered_at фіксується при першому переході в Доставлено                                                                                                                 
  - delivered_at не перезаписується при повторному Доставлено                                                                                                                 
  - assign_deliveries — призначає кур'єра та змінює статус на Розподілено                                                                                                     
  - assign_deliveries — ключ unassigned скидає статус на Очікує                                                                                                               
  - assign_deliveries — доставки поза переданим списком не чіпаються                                                                                                          
                                                                                                                                                                              
  OrderService                                                                                                                                                                
                                                                                                                                                                              
  - Створення разового замовлення → рівно 1 Order + 1 Delivery                                                                                                                
  - Дата доставки в Delivery = даті з форми
  - is_pickup = on → delivery.street = None                                                                                                                                   
  - custom_amount приходить як рядок '500' → зберігається як int 500                                                                                                          
  - Невідомий instagram в get_or_create_client → (None, помилка)                                                                                                              
  - Відомий instagram → повертає клієнта                                                                                                                                      
  - Фільтр delivery_type='One-time' → повертає тільки разові замовлення (без subscription_id)                                                                                 
  - (xfail BUG-03) Дата в минулому має відхилятись — наразі не валідується                                                                                                    
  - (xfail BUG-04) time_from > time_to має відхилятись — наразі не валідується                                                                                                
                                                                                                                                                                              
  Certificate model                                                                                                                                                           
                                                                                                                                                                              
  - effective_status() = active коли не прострочено                                                                                                                           
  - effective_status() = expired коли дата минула
  - effective_status() = used незалежно від дати                                                                                                                              
  - Сертифікат з expires_at = сьогодні → active (не прострочений)                                                                                                             
  - is_active() = True коли активний і не прострочений                                                                                                                        
  - is_active() = False коли прострочений                                                                                                                                     
  - is_active() = False коли used                                                                                                                                             
  - is_active() завжди відповідає effective_status() == 'active'
